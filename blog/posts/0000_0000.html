关联类型 ​

关联类型（associated types） 让我们可以在 trait 里面增加一个待定义的类型（类型占位符），将类型占位符与 trait 相关联，这样 trait 的方法签名中就可以使用这些占位符类型。

关联类型与泛型的区别 ​

使用关联类型的 trait 只允许为同一种类型实现一次，而泛型 trait 则可以为同一种类型实现多次。

rust
struct Counter {
    index: u32
}

// 使用关联对象的trait
trait ATrait {
    type Item;

    fn next(&mut self) -> Option<Self::Item>;
}

// 使用泛型的trait
trait BTrait<T> {
    fn next(&mut self) -> Option<T>;
}


// 一次实现
impl ATrait for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {}
}

// 可根据不同泛型参数多次实现
impl BTrait<i32> for Counter {...}

impl BTrait<f32> for Counter {...}

impl BTrait<u32> for Counter {...}

关联类型也会成为 trait 契约的一部分：trait 的实现必须提供一个类型来替代关联类型占位符。关联类型通常以它的用途来命名，并且我们最好在 API 文档中为关联类型编写文档。

默认泛型参数 ​

当使用泛型类型参数时，可以为泛型指定一个默认的具体类型。如果默认类型就足够的话，这消除了为具体类型实现 trait 的需要。

而这项技术在运算符重载时发挥重要作用，这里以 + 运算符重载为例，下面为 Add trait 的定义。

rust
trait Add<Rhs=Self> {
    type Output;

    fn add(self, rhs: Rhs) -> Self::Output;
}

可见，泛型参数 Rhs 的默认类型为 Self。

因此，当加号运算符两侧都为同一种类型时，泛型参数可以省略。

当需求为两种不同类型的变量相加，也就是 Rhs 类型不为 Self 时，我们依旧需要显式地指定泛型参数。

rust
struct Meter(u32);
struct Mile(u32);

// 实现 Meter = Meter + Meter
impl Add for Meter {
    type Output = Meter;
    
    fn add(self, other: Meter) -> Self::Output {
        Meter(self.0 + other.0)
    }
}

// 实现 Meter = Meter + Mile
impl Add<Mile> for Meter {
    type Output = Meter;

    fn add(self, other: Mile) -> Self::Output {
        Meter(self.0 + other.0)
    }
}

判断是否使用默认泛型参数，可以参考以下两点：

  • 在大部分用户都不需要的特定情况进行自定义。
  • 扩展类型而不破坏现有代码。

完全限定语法 ​

当类型自身与 trait 拥有同名方法，或实现的多个 trait 有同名方法，下方代码展示了如何正确地调用它们。

rust
struct Window;

trait Widget { fn draw(&self); }

trait Movable { fn draw(&self); }

impl Window {
    fn draw(&self) {
        println!("this is draw from Window!");
    }
}

impl Widget for Window {
    fn draw(&self) {
        println!("this is draw from Widget");
    }
}
impl Movable for Window {
    fn draw(&self) {
        println!("this is draw from Movable");
    }
}

fn main() {
    let object = Window;
    object.draw(); // this is draw from Window!
    Widget::draw(&object); // this is draw from Widget!
    Movable::draw(&object); // this is draw from Movable!
}

注意

存在同名方法时，Rust 将优先调用直接实现在类型上的方法。

上述方式中，编译器通过 方法（method） 的第一个参数self的类型来做出正确调用。

如果在相同情景下，同名的不是方法而是函数，由于函数没有self参数用于推测类型信息，我们需要使用 完全限定语法。

rust
trait Widget { fn draw(); }
trait Movable { fn draw(); }
...

impl Window {
    fn draw() { ... }
}
...

fn main() {
    Window::draw();
    <Window as Widget>::draw(); 
    <Window as Movable>::draw(); 
}

父 trait ​

当一个自定义 trait TraitA 依赖于另一个 trait TraitB 的某些功能（方法）时，我们可以把 traitB 作为 traitA 的 父 trait。

rust
trait TraitA: TraitB {
    ...
}

最常见的例子就是 Clone 是 Copy 的父trait。

孤儿规则及特例 ​

孤儿规则（orphan rule） 的定义为：只要满足 trait 或 类型 其中之一定义在本地的 crate，就允许为此类型实现这个 trait。例如，无法为u32类型实现Display trait。

但孤儿规则可以通过 newtype模式 绕过。同时在以下三种情况下此规则将不再适用：

  • 基础类型
  • Blanket Implementation
  • Covered Implementation

newtype 模式 ​

使用 newtype模式 允许我们为外部类型实现本地 trait。其本质是将外部类型用元组结构体包装为本地类型，从而绕过孤儿规则。示例：

rust
use std::fmt::Display;

struct Int(i32);

impl Dispaly for Int { ... }

基础类型 ​

对于过于基础的类型，Rust 允许任何人在它们上实现 trait，即使违反孤儿规则。

基础类型 被#[fundamental]标记，它们包括：

  • 不可变引用 &
  • 可变引用 &mut
  • 智能指针 Box<T>

Blanket Implementation ​

Blanket Implementation 允许为所有实现了某个 trait 的类型实现这个 trait，而非某个特定类型。其格式为：

rust
impl<T> MyTrait for T
    where T: ...
{
    ...
}
rust
// 为所有实现了 Display trait 的类型实现 MyTrait
impl<T: Display> MyTrait for T {
    ...
}

注意

只有在定义 trait 的 crate 里才允许使用 Blanket Implementation。

Covered Implementation ​

对于极少数需要为外部类型实现外部 trait 的情况，孤儿规则制定了 Covered Implementation 这一豁免条件来满足需求。

rust
impl<P1..=Pn> ForeignTrait<T1..=Tn> for T0 {
    ...
}

上述写法只在以下条件均满足时被 Rust 允许：

  • 至少有一个 Ti 是本地类型。这里的 Ti 为 T1..Tn 中的一个。
  • 没有未被 cover 的 T 出现在第一个这样的 Ti 之前。这里的 T 为 P1..Pn 中的一个。

需要注意的是，这里的 cover 定义如下：

  • 如果 T 作为其他类型的泛型参数出现 （如 Vec<T>），那么就称 T 被 cover 了。
  • 但是 T 本身和其 基础类型（如&mut T）不是 Cover。

例:

rust
// Ok
impl<T> From<MyType, T> for Vec<T>

// Error
impl<T> From<MyType, T> for T // 不满足条件二：T0 中未 cover 的 T 出现在 MyType 前
impl<T> From<T, MyType> for Vec<T> // 不满足条件二：T 出现在 MyType 前
impl<T> From<MyType<T>> for T // 不满足条件二：T 出现在 MyType 前
impl<T> From<T> for Vec<T> // 不满足条件一
