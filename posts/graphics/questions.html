图形学解疑 ​

本章是笔者计算机图形学学习过程中出现的疑惑点的解疑合集。

Gamma 矫正和 sRGB 图像 ​

本节将解释使用 Gamma 矫正和 sRGB 图像的原因。

这里首先展示渲染流程中 颜色空间 的变化过程作为前置知识：

text
纹理图像(sRGB) -> 图形 API 矫正 -> 光照计算  -> Gamma 矫正  -> 显示器显示
----------------------------------------------------------------------
   sRGB空间         线性空间       线性空间     Gamma空间      线性空间

1. 为什么需要 Gamma 矫正？ ​

由于物理特性和历史遗留，直到现在显示器仍然保留着这样的特性： 将输入亮度（电压）加倍产生的亮度约为输入亮度（电压）的2.2次幂。2.2被称为显示器 Gamma 值。

Gamma 矫正 提前将颜色从线性空间映射到 Gamma 空间，再由显示器重新映射回线性空间。

  • Gamma 矫正
I2=I112.2
  • 显示器映射
I3=I22.2
  • 显示结果
⇒I3=I1

此过程确保显示亮度和光照计算结果保持线性一致。

2. 为什么需要 sRGB 图像？ ​

使用 sRGB 图像标准 存在两个明显好处：便利图像创作和提高图像存储效率。

  1. 在使用计算机创建图像时，我们通常根据显示器画面来调整颜色，使其更加符合视觉效果。

    例如，在软件中通过调色使一张风景照片显示出来的亮度更符合人眼在现实中感受到的。

    这样得到的图像正是 sRGB 图像：由于显示器 Gamma 的存在，我们在调色过程中无意地对图像进行了一次 Gamma 矫正来达到正确的显示效果。

  2. 根据韦伯定律，人眼感受到的物理亮度要增加一倍（比如从0.1到0.2）才会感觉比原来变亮了一倍。

    例如，颜色值从0.1到0.2，我们会感受到一倍的颜色变化，而从0.4到0.8我们才能感受到相同程度（变亮一倍）的颜色变化！

    而对数函数似乎能完美地描述物理亮度和视觉亮度的关系：

    ln(0.2)−ln(0.1)=ln2=ln(0.8)−ln(0.4)

    自然地，我们会选用 f(x)=lnx+1 来描述两者关系，观察图像：

    （其中横坐标表示物理亮度，纵坐标表示感知亮度）

    可以发现，物理亮度的暗部（0<x<0.5）就已经能映射到视觉亮度的绝大部分区域（阴影部分下边界向下无限延伸）。

    这意味着人眼对现实光线中的暗部有着更大的感知范围，即人眼对暗部变化更加敏感！（相应地，人眼对亮部的变化更不敏感。）

    因此，在图像内存占用不变的情况下，我们希望牺牲部分亮度信息，以此保留更多暗部信息。

    好巧不巧，Gamma 矫正对应的指数函数 f(x)=x12.2 正好能够将暗部映射到更大的范围上去，为暗部信息保留了更多存储空间。

    所以，存储Gamma矫正后的图像是更加合理的选择：暗部信息被分配到更多的bit去存储。代价是在用于光照计算前需要矫正回线性空间。

    而所谓的“Gamma矫正后的图像”正是 sRGB 图像。

    关于物理亮度和感知亮度的关系曲线

    实际上物理亮度和感知亮度的关系遵循一个更加复杂的函数，而不能简单地用对数函数描述。

    以下是一个简单证明：

    首先，只有对数函数满足特征：

    f(ax)−f(x)=lna=c

    并且任何关于 x 轴的平移操作将会破坏这个特征：

    f(ax+b)−f(x+b)=lnax+bx+b≠c

    这意味着 f(x) 是未经水平平移的对数函数。

    然而，根据现实中的亮度感知，函数还需满足 f(0)=0。

    显然，无水平平移的对数函数无法满足上述条件，矛盾。

法线矩阵的推导 ​

先考虑经模型矩阵变换后的切向量。

平面切向量可以表示为面上两点之差：

T⇀=pa−pb

将点 pa 和 pb 经模型矩阵变换后，根据切线定义，它们之差仍表示该平面的切向量。

T⇀′=Mpa−Mpb=M(pa−pb)=MT⇀

因此，经模型变换后的切向量 即为模型矩阵乘以原切向量：

(1)T⇀′=MT⇀

接下来，我们开始推导法线矩阵 G。它的作用是正确地将法向量进行模型变换：

(2)N⇀′=GN⇀

变换后，由于法向量和切向量始终仍保持垂直，两者内积为 0：

dot(N⇀′,T⇀′)=0

将式 (1)(2) 代入，得到：

dot(GN⇀,MT⇀)=0(GN⇀)T(MT⇀)=0(N⇀TGT)(MT⇀)=0(3)N⇀TGTMT⇀=0

变换前，法向量和切向量保持垂直，两者内积为 0：

(4)dot(N⇀,T⇀)=N⇀TT⇀=0

由 (3)(4) 可知：

GTM=IGTMM−1=M−1GT=M−1(5)G=(M−1)T

式 (5)： G=(M−1)T 即为法线矩阵与模型矩阵的转换关系。

TBN 矩阵的推导及应用 ​

本节将介绍 TBN 矩阵的推导以及实际应用时的细节。

推导 ​

由于直接从法线贴图中采样得到的法向量定义在切线空间中，我们需要使用 TBN 矩阵将其变换到世界空间下以参与光照计算。

切线空间（Tangent Space）

切线空间是假定位于平面上方的空间。它可以由 (T,B,N) 三个单位正交向量作为基底定义，要求 T 和 B 分别与纹理坐标系的 U 和 V 同向，N 垂直于平面向外。

因此，要想找到切线空间下的法向量在世界空间中的表示，只需找到平面 (T,B,N) 的世界空间表示并将其作为新基底即可，向量表示为：

N⇀world=NtanxT⇀+NtanyB⇀+NtanzN⇀

TBN 矩阵即为上述变换的矩阵形式。

由于 N 向量通常由模型提供或通过 T×B 得到，因此我们只推导 T 和 B 即可。

假定 P1,P2,P3 为平面上的三个顶点，得到下图：

由图可知 E1,E2 可以表示为 T 和 B 的线性组合：

E⇀1=ΔU1T⇀+ΔV1B⇀E⇀2=ΔU2T⇀+ΔV2B⇀

为了写成矩阵形式，我们将上式的向量用分量展开：

(E1x,E1y,E1z)=ΔU1(Tx,Ty,Tz)+ΔV1(Bx,By,Bz)(E2x,E2y,E2z)=ΔU2(Tx,Ty,Tz)+ΔV2(Bx,By,Bz)

因此，矩阵形式即为：

[E1xE1yE1zE2xE2yE2z]=[ΔU1ΔV1ΔU2ΔV2][TxTyTzBxByBz]

为了分离出 (T,B) 矩阵，两侧左乘一次逆矩阵：

[ΔU1ΔV1ΔU2ΔV2]−1[E1xE1yE1zE2xE2yE2z]=[TxTyTzBxByBz]

使用伴随矩阵法算出逆矩阵，整理后可得到 (T,B) 矩阵表示：

[TxTyTzBxByBz]=1ΔU1ΔV2−ΔU2ΔV1[ΔV2−ΔV1−ΔU2ΔU1][E1xE1yE1zE2xE2yE2z]

应用 ​

在实际编写着色器时，由于 N 通常已经由模型作为顶点数据提供，我们只需要计算 T 或 B 其中之一，然后用叉乘得到另一个向量。

以优先计算 T 为例，根据上方推导结果 T 可单独表示为：

[TxTyTz]=1ΔU1ΔV2−ΔU2ΔV1[ΔV2−ΔV1][E1xE1yE1zE2xE2yE2z]

由于计算得到的向量总需要被单位化，因此可以忽略常数项：

[TxTyTz]=[ΔV2−ΔV1][E1xE1yE1zE2xE2yE2z]

转化为向量表示：

T⇀=ΔV2E⇀1−ΔV1E⇀2

综上，最终 T,B,N 三个向量（未单位化）可表示为：

N⇀=N⇀givenT⇀=ΔV2E⇀1−ΔV1E⇀2B⇀=T⇀×N⇀

在实现过程中，常用着色器语言提供的偏导数函数 dFdx 与 dFdy 获取 E 和 ΔV。

GLSL 的具体实现可以参考 PBR.shader。
