Deref Trait 解析 ​

rust
trait Deref {
    type Target: ?Sized;
    fn deref(&self) -> &Self::Target;
}

Deref 特征参与了非引用到引用转换 T -> &U，以及引用到引用转换 &T -> &U。

前一类表现在为自定义类型添加解引用操作支持，例如 Box<T>。 后一类表现在 Deref 强制转换，例如 &String -> &str 和嵌套引用的自动解引用 &&T -> &T。 无论如何，正如特征名 Deref 所表达的，两种行为都在通过 deref“脱壳”来获取被包装的内部类型。

巧妙的是，Deref::deref 方法调用通常在编译期自动生成。不巧的是，我们无法直观观察引用的连续转化过程。

因此，本文将首先分析 deref 调用方式，并以此来解析上述两类转换的deref过程。

deref 方法调用 ​

需要明确的是：在调用 trait 以及类型方法时，编译器可能会进行隐式取引用来匹配参数类型。

例如在下方例子中，access 方法能在 Data 以及 &Data 上调用。

rust
struct Data(i32);

impl Data {
    #[allow(dead_code)]
    fn access(&self) {}
}

fn main() {
    let data = Data(1);
    let r = &data;

    data.access();
    r.access();
}
