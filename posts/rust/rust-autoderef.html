Rust 隐式转换与自动解引用 ​

Deref 引用参数的匹配规则 ​

Deref 或 Copy + Clone 这类影响类型行为的 trait，其关联方法常常由编译器隐式调用，因此想要探究具体调用行为比较困难，其中之一就是引用参数的传递方式。

因此在多次实践验证后，我总结了编译器在调用此类方法时传递引用参数的规则：符合参数类型的直接传入，否则取引用后传入。

实际上，接下来的即将解释的两个现象都利用了此规则。

以下是例子，其中显隐式调用等价：

  • 直接传入：&String -> &str
rust
let name = String::from("Jack");
let r1: &String = &name;

// 隐式调用
let r1: &str = r1;

// 显式调用
let r1: &str = Deref::deref(r1);
  • 取引用后传入：*Box<T> -> T
rust
let data = Box::new(1);

// 隐式调用
let num = *data;

// 显式调用
let num = *(Deref::deref(&data));

引用可变性的隐式转换 ​

众所周知，可变引用可以隐式转换为不可变引用，即 &mut T -> &T。这个转换是由 Deref 实现的：

rust
impl Deref for &mut T {
    type Target = T;

    fn deref(self: &&mut T) -> &T {
        *self
    }
}
rust
let mut data = 1.0;
let r1 = &mut data;

// 隐式调用
let r2: &f64 = r1;

// 显式调用
let r2: &f64 = Deref::deref(&r1);

嵌套引用的自动解引用 ​

Rust 为了简化简化嵌套引用的使用，为其实现了自动解引用的功能。而十分巧妙的是，可变性隐式转换和嵌套引用自动解引用两个功能都是由 Deref 实现，也就是说一套代码实现了两个功能！

以不可变引用 &&T 为例：

rust
impl Deref for &T {
    type Target = T;

    fn deref(self: &&T) -> &T {
        *self
    }
}
rust
let data = 123;
let r1: &i32 = &data;
let r2: &&i32 = &r1;

// 隐式调用
let r: &i32 = r2;

// 显式调用
let r: &i32 = Deref::deref(r2);

对于譬如 &&&&T 这样更多层的嵌套引用，编译器会连续deref直到只剩下一层引用 &T。

可变引用则用 DerefMut 实现了该功能，原理同上。
